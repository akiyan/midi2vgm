# GG 向け PSG 化マッピング（scripts/genallpsg.py が読む）。
#
# SN76489（矩形3ch + ノイズ1ch）に MIDI のどのパートを当てるかを曲ごとに手動指定する。
# [曲ファイル名（拡張子なし）] のセクションごとに以下の行を書く:
#
#   tone0 = <MIDI ch> [decay=pluck|soft|sustain] [oct=±N] [vib] [atten=N]
#   tone1 = ...                # 矩形2本目
#   tone2 = ...                # 矩形3本目（多くはベース。PSG は低音が出ないので oct=+1 で持ち上げる）
#   noise = drums              # ドラム(ch10)をノイズへ。書かなければノイズ無し
#
#   MIDI ch は 1 始まり（midi_roles の表示と同じ）。tone は単音化（最後に押した音を優先）。
#   メロディが複数パートに分かれて受け渡される曲は ch を '+' で連結して 1 本に共有できる
#   （例: tone0 = 1+5）。共有時は「最高音」優先で、片方が休んでも他方で繋いで無音を防ぐ。
#   decay: pluck=速い減衰 / soft=ゆるい減衰(既定) / sustain=減衰なし(note-off まで保持)
#   oct=+1 等で全体をオクターブ移調。範囲外(約 A2〜C7)は自動でオクターブ折り返し。
#   vib でソフトビブラート（持続するリード向け）。atten=N で更に N 段音量を下げる。
#
# セクションの無い曲は自動選定（bass + 音符数上位2、tone0=高音…tone2=低音）。
